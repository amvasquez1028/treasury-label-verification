# Content style

Follow plain-language principles aligned with USWDS guidance:

- Use active voice: "Upload a label image" not "A label image should be uploaded"
- Lead with the action or outcome in errors: "Select a label image to verify."
- Avoid jargon where possible; spell out ABV on first reference in docs
- Status labels: **Verified** / **Not verified** — not "Success/Fail" alone
- Loading states end with ellipsis: "Signing in…", "Verifying session…"

## Error tone

Be specific and actionable. Include what the user can do next without exposing stack traces or internal IDs.
